# 与 AI 协作完成项目的过程说明

## 1. 说明

本项目在 2026 年 4 月 17 日至 2026 年 4 月 18 日的开发过程中，使用了 AI 工具协助完成方案设计、代码实现、测试验证和文档整理。  
本项目主要使用的 AI 工具是 `Codex`。

这份文档用于简要说明：在整个项目推进过程中，哪些工作由 AI 协助完成，哪些关键判断由人工提出和决定，以及双方如何通过多轮迭代逐步收敛出当前版本。

## 2. AI 参与的主要工作

AI 在本项目中主要承担了以下工作：

- 根据 brief 讨论并整理需求
- 输出 requirement 1、requirement 2、requirement 3 的文档初稿
- 设计并实现 Python 版本的钱包 MCP Server 项目骨架
- 生成并迭代钱包相关工具，包括查询、提案、确认、执行、历史、收款信息等功能
- 设计并实现 Streamlit 版 Operator Console Demo
- 协助补充白名单、地址簿、交易历史、提案管理等功能
- 协助检查权限边界、执行流程和风控逻辑
- 进行本地模拟测试、回归测试和问题修复
- 根据人工反馈反复精简文档表达与页面展示

可以理解为，AI 在本项目中承担了“实现助手”和“文档助手”的角色。

## 3. 人工主导的关键决策

虽然 AI 参与了大量实现工作，但项目中的关键方向和约束主要由人工提出并不断修正。主要包括：

- 明确项目应以 `Python` 实现，而不是其他语言
- 明确项目应是 `Codex 可调用的 MCP Server 钱包 Demo`
- 明确当前阶段优先完成 `本地模拟`，而不急于做真实链上广播
- 要求转账流程采用 `proposal -> 用户确认 -> execute transfer` 的模式
- 决定默认不采用本地 PIN 二次授权，而是保留为可选严格模式
- 要求敏感配置不能交给 Agent，而应放到人工后台页面处理
- 决定增加 Streamlit 版网页后台，用于管理私钥、白名单、地址簿、模拟余额和策略开关
- 要求地址支持联系人命名化，而不是只接受完整地址
- 要求增加白名单机制，并不断检查白名单限制是否真正生效
- 要求增加交易历史、提案取消、账户信息查询、收款信息展示等能力
- 对文档结构、页面文案、命名和展示风格进行多次人工纠正和简化

也就是说，AI 负责快速产出和修改实现，但项目的产品取舍和安全边界主要由人工决定。

## 4. 协作推进过程

### 4.1 需求理解阶段

项目开始时，先围绕项目大致要求讨论了目标：做一个供 AI Agent 使用的加密货币钱包 Demo。  
在这一阶段，AI 主要负责把分散的想法整理成可执行的架构和文档草稿。

### 4.2 架构设计阶段

在架构阶段，AI 先给出初版架构文档与功能拆分，之后根据人工反馈持续调整，不断压缩、修正和限定范围。

### 4.3 代码实现阶段

在实现阶段，AI 按照项目讨论结果逐步完成了：

- MCP Server 主体
- 钱包工具层
- Proposal 状态机
- 本地模拟执行
- 交易历史与提案查询
- 地址簿与白名单支持
- 收款信息展示
- Operator Console

这部分工作以“先搭骨架，再逐步补功能”的方式推进，每次都由人工提出新增能力或调整流程，AI 再完成对应修改。

### 4.4 测试与修复阶段

项目后期，大量协作集中在测试和修复问题上。人工多次直接指出真实使用中的异常，例如：

- 转账前没有先做余额校验
- proposal 被提前消费
- 白名单开启后仍可向非白名单地址转账
- Operator Console 修改策略后，MCP 侧没有及时同步
- 权限门卫没有正确拦截某些请求
- 金额和手续费显示小数位不合理
- 页面展示不清晰
- `intent_hash` 看起来存在，但实际执行前未形成完整校验

AI 根据这些问题逐步补充和修正实现。  

## 5. 本次协作的特点

本项目中，AI 并不是一次性生成最终结果，而是通过多轮对话不断迭代。  
整体过程更接近下面这种协作模式：

1. 人工提出目标或指出问题
2. AI 给出方案、代码或文档初稿
3. 人工检查并指出偏差、风险或不符合要求的地方
4. AI 根据反馈继续修改
5. 重复上述过程，直到收敛到可接受版本

## 6. 结果总结

通过这次协作，AI 工具主要帮助项目提升了以下效率：

- 更快完成项目骨架搭建
- 更快生成和修改重复性代码
- 更快产出文档初稿并做格式整理
- 更快完成回归测试和问题定位

而人工在整个过程中主要保证了以下事情：

- 项目方向不偏离项目要求
- 安全边界和交互流程符合预期
- 文档与页面表达更贴合作业提交要求
- 关键风险不会被“看起来能跑”所掩盖

因此，本项目最终成果应理解为：在人工主导需求和审查的前提下，借助 AI 工具共同完成的一个 AI Agent 钱包 Demo。
